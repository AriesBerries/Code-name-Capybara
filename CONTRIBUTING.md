# Contributing to Code name Capybara

Thank you for your interest in contributing!

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a feature branch
4. Make your changes
5. Submit a pull request

## Development Guidelines

### Code Standards
- **Elm**: Follow elm-format standards
- **Go**: Follow gofmt and golint
- **Rust**: Follow rustfmt and clippy

### Documentation
- Update relevant artifacts in `docs/artifacts/`
- Add examples and diagrams where helpful
- Keep terminology consistent with [02-TERMINOLOGY_GLOSSARY.md](docs/artifacts/ARTIFACT_02_TERMINOLOGY_GLOSSARY.md)

### Testing
- Write unit tests for all new features
- Ensure integration tests pass
- Update E2E tests if UI changes

## Pull Request Process

1. Update documentation to reflect changes
2. Ensure all tests pass
3. Update CHANGELOG.md
4. Request review from maintainers
5. Address feedback
6. Merge after approval

## Community Guidelines

- Be respectful and inclusive
- Follow our Code of Conduct
- Ask questions - no question is too small
- Help others when you can

## Questions?

Open an issue or reach out to the maintainers.
